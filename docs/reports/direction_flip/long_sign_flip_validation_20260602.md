## 7. コードレビュー: US→JP 写像 正負ロジック

### コードレビュー: US→JP 写像の正負ロジック

#### 1. 固有ベクトル (prior.py:7-27)

```
v1 = normalize(ones(n))               # グローバル因子: US/JP 同方向
v2_raw = [+1, ..., +1, -1, ..., -1]  # 国別スプレッド: US正, JP負
v3 = cyclical_defensive              # シクリカル/ディフェンシブ
```

**v2 の符号**: US = +1, JP = -1。
→ `c0` はこの prior で正則化されるため、v2 成分では US と JP が **逆符号** の固有ベクトルが生成される。

#### 2. シグナル計算 (model.py:12-31)

```python
v_k = eigvecs[:, -k:]      # 固有値昇順のため -k: が最大k固有ベクトル
v_u = v_k[:n_u, :]         # US factor loadings (shape: n_u × k)
v_j = v_k[n_u:, :]         # JP factor loadings (shape: n_j × k)
f_t = v_u.T @ z_us_t       # US factor scores   (shape: k,)
signal = v_j @ f_t          # JP signal          (shape: n_j,)
```

**符号の流れ**:
`signal_j = Σ_k (v_jk × Σ_u v_uk × z_us_u)`

- **グローバル因子 (v1 型)**: US/JP 同符号 → US が上がると JP signal 正 → LONG JP
- **国別スプレッド因子 (v2 型)**: US 正, JP 負 → US が上がると JP signal 負 → SHORT JP

#### 3. 符号反転の根本原因

`c_reg = (1-λ)*c_t + λ*c0` で正則化後の固有分解において、
`v2` 型の「US 上昇 → JP 下落」方向の因子が支配的なとき:

- `f_t > 0`（US が上がった）
- `v_j` が v2 成分では **負** → `signal_j < 0`
- にもかかわらずコードは `signal ≥ 0.10` → **LONG** に分類

→ これは「US が上がった時に JP が下がりやすいセクター」をLONGに入れる設計になっている可能性。

**ただし**、グローバル因子 (v1 型) が支配すれば US 上昇 → JP signal 正 → LONG は正常。
実際のシグナルは k=3 の混合であり、どの因子が支配的かは period・regime によって変わる。

#### 4. バックフィル OC vs CC horizon の比較から得られた知見

- **CC 2年**: 現行方向 +127.52% vs LONG反転 +27.72% → **CC では現行方向が圧倒的に正しい**
- **OC 2年**: 現行方向 +30.01% vs LONG反転 +87.90% → OC では LONG反転が優位
- **CC 1年**: 現行方向 +8.36% vs LONG反転 -10.41% → CC では現行が有効
- **OC 1年**: 現行方向 -6.47% vs LONG反転 +32.94% → OC では LONG反転が優位

この OC/CC の逆転は **執行 horizon 不整合** を示す:

1. US シグナルは JP の翌日 **終値（CC）** を正しく予測している
2. JP 市場は寄り付き時に US シグナルを先取り（ギャップアップ）する
3. 寄り付き後はイントラデイで平均回帰（OC はマイナス）
4. 現行の OC 執行（寄り付き買い → 引け売り）はギャップ後リバーサルで損をしている

→ **シグナルの符号は正しい。問題は OC 執行タイミング。**

- signal × next_oc (LONG群): 全期間で **負** だが、2年では小幅 (-0.000163)、直近3ヶ月では大きく負 (-0.001189)

#### 5. 実装上のバグの有無

- `eigh(...) -> eigvecs[:, -k:]`: **正しい**（scipy.linalg.eigh は昇順）
- `v2_raw = [+1]*n_u + [-1]*n_j`: **意図通り**（US/JP 逆符号）
- `gram_schmidt` の直交化: **正しい**
- `signal = v_j @ f_t`: **正しい計算**、ただし v2 因子の支配時に符号が逆になるのは設計の本質的性質

#### 6. 結論（コードレビュー）

**実装バグはない。シグナルの方向自体も CC 基準では正しい。**

主因は OC 執行のホライズン不整合。
US クローズ → JP ギャップアップ（US 情報織り込み）→ JP イントラデイ平均回帰、というフローが
現行の「寄り付き買い → 引け売り」執行に対して逆風になっている。

修正優先度:
1. **執行タイミングを見直す**（CC 基準への移行 or ギャップ捕捉）
2. シグナルの符号を反転しない — CC では現行方向が有効
3. regime による方向動的切替は CC 移行後に再評価

## 8. 結論

### 判定: Pattern C

**LONG 反転が OC のみ改善、CC では現行が有効 → 執行 horizon 不整合**

OC (翌日 Open→Close) 基準では LONG 反転が全期間で優位だが、
CC (前日終値→当日終値) 基準では現行方向が長期で正しく機能している。

| 基準 | 現行 2年 | LONG反転 2年 |
| --- | --- | --- |
| OC | +30.010% | +87.898% |
| CC | +127.519% | +27.720% |

**解釈**: US シグナルは JP の翌日終値（CC）を正しく予測している。
しかし JP 市場は寄り付き時に US シグナルを先取り（ギャップアップ）し、
その後イントラデイ（OC）で平均回帰する。
